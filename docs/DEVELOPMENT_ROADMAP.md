# 테일즈오브발레리아 (Tales of Valeria) — 개발 로드맵

> 1인 + AI 에이전트 개발 기준.
> 게임 개발 경험 없이 시작. 단계별로 검증하며 진행.

---

## 게임 개발의 큰 흐름

```
기획 → 프로토타입 → 알파 → 베타 → 출시
```

각 단계는 이전 단계가 완성되어야 넘어가는 게 아니라,
**핵심 루프가 재미있는지 검증**하며 점진적으로 확장하는 방식.

---

## 단계별 목표

### 0단계: 기획 (현재 ✅)
> "무엇을 만들 것인가"를 결정하는 단계.

- [x] 세계관, 분위기, 장르 확정
- [x] 핵심 철학 ("사랑이 승리한다")
- [x] 엔진 선택 (Unity)
- [x] 캐릭터·성장·역할 시스템 설계
- [x] 세계 구성 (5개 권역)
- [x] 2차 역할 조직 및 스킬 설계
- [ ] 7대 역할 스킬 트리 상세 설계 ← 구현 직전에 해도 됨

---

### 1단계: 환경 세팅
> "개발 도구를 준비하는 단계."

- [ ] Unity 설치 및 프로젝트 생성 ← 진행 중 (사용자 수동)
- [x] 아이소메트릭 카메라 세팅 (IsometricCamera.cs 작성 완료)
- [x] 기본 폴더 구조 정리 (Assets/Scripts 등 생성 완료)
- [x] Git 연동 + .gitignore 설정
- [ ] 무료 아이소메트릭 에셋 찾아서 테스트 환경 구성

**목표**: 화면에 뭔가 보이면 성공.

---

### 2단계: 프로토타입 (가장 중요한 단계)
> "핵심 게임플레이가 재미있는지 검증."
> 그래픽은 임시, 기능만 작동하면 됨.

- [ ] 클릭 이동 (마우스 클릭 → 캐릭터 이동)
- [ ] 기본 적 AI (캐릭터 감지 → 추적 → 공격)
- [ ] 기본 전투 (클릭으로 적 공격, HP 감소, 사망)
- [ ] 카메라 추적

**목표**: 클릭해서 이동하고, 적과 싸울 수 있으면 성공.
**검증**: "이게 일렌시아처럼 느껴지는가?"

---

### 3단계: 핵심 시스템 구현
> "발레리아만의 특징적인 시스템 구현."

- [ ] 숙련도 시스템 (스킬 사용 → 숙련도 누적)
- [ ] 7대 역할 자동 배정 로직
- [ ] 기본 스킬 트리 (역할 1~2개만 먼저)
- [ ] 인벤토리 / 아이템 시스템
- [ ] 기본 NPC 대화 시스템
- [ ] 용병 길드 UI (의뢰 수주)

**목표**: 플레이하면서 역할이 바뀌는 느낌이 나면 성공.

---

### 4단계: 월드 구축
> "실제 게임 공간을 만드는 단계."

- [ ] 수도권 맵 제작 (첫 번째 권역)
- [ ] 기본 퀘스트 1~2개
- [ ] 주요 NPC 배치 (왕궁, 아크론교, 용병 길드)
- [ ] 나머지 4개 권역 순차 제작
- [ ] 권역 간 이동 경로 + 필드 몬스터

**목표**: 한 권역을 1~2시간 플레이할 수 있으면 성공.

---

### 5단계: 알파
> "전체 윤곽이 갖춰지는 단계. 내부 테스트."

- [ ] 5개 권역 전부 플레이 가능
- [ ] 2차 역할 조직 시스템 구현
- [ ] 메인 스토리 퀘스트 라인 (뼈대)
- [ ] 스포라교 은유 요소 배치
- [ ] 부유성 클라이맥스 구현

**목표**: 처음부터 끝까지 스토리를 클리어할 수 있으면 성공.

---

### 6단계: 베타
> "완성도를 높이는 단계. 외부 테스트."

- [ ] 사운드 & 음악
- [ ] 버그 수정 및 밸런스 조정
- [ ] 외부 플레이어 테스트 피드백 반영

### 7단계: 출시 준비

## 핵심 원칙

### 재미 우선
> 그래픽, 완성도보다 "핵심 루프가 재미있는가"를 먼저 검증.
> 프로토타입이 재미없으면 기획을 수정하는 게 낫다.

### 점진적 확장
> 처음부터 전부 만들지 않는다.
> 수도권 하나 → 재미있으면 → 다음 권역.
> 역할 1개 → 재미있으면 → 7개 전부.

### 기획은 유연하게
> 구현하다 보면 기획이 바뀐다. 이건 정상이다.
> 문서는 참고용이지 고정된 계약서가 아니다.

### AI 에이전트 활용
> Unity C# 코드는 AI가 생성.
> 개발자의 역할은 "무엇을 만들 것인가"를 결정하고,
> AI가 만든 결과물이 의도대로 작동하는지 검증하는 것.
